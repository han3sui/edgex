# Modbus 驱动深度优化验证清单

## 1. 核心连接管理 (Transport Layer)
- [ ] **长连接复用**: 确保一个 Channel 只有一个 TCP 连接，所有设备通过 SlaveID 切换复用。
- [ ] **退避重连**: 模拟断网，验证重连间隔是否符合 1s -> 2s -> 4s -> ... 300s。
- [x] **心跳检测**: 验证 30s 周期性心跳，3 次失败后是否触发断开重连。
  - ✅ 已实现工业Modbus TCP采集惯例：一个串口下多个设备，只要有**一个**设备正常应答报文，则认为会话正常，无需断开TCP连接
  - ✅ 新增 `sessionTimeout` 配置（默认90s）：会话超时时间，只要在此时间内有成功通信即认为会话健康
  - ✅ 新增 `heartbeatFailMax` 配置（默认3）：最大心跳失败次数
  - ✅ 协议错误（illegal/exception/busy/CRC）不会断开TCP连接，只有网络/IO错误才触发重连

## 2. 智能探测与优化 (Intelligent Probing)
- [ ] **MTU 探测**: 验证启动后是否自动探测并设置了合理的 MaxBatchSize。
- [ ] **无效地址跳过**: 配置无效地址，验证驱动是否自动标记并跳过，不再频繁重试。

## 3. 调度器增强 (Scheduler Enhancement)
- [ ] **AIMD 算法**: 在稳定网络下 BatchSize 增加，丢包环境下 BatchSize 减少。
- [ ] **自适应间隔**: 采集耗时增加时，下次采集间隔自动延长，保持 30-50% 负载。

## 4. 高性能解码 (Zero-Copy Decoder)
- [ ] **全格式覆盖**: 验证 16 种 Modbus 数据格式（Float/Double/Long）解码正确无误。
- [ ] **性能基准**: 确保新解码器比老版本快 > 30%（Benchmark）。

## 5. 数据缓冲与提交 (Data Buffering)
- [ ] **无数据丢失**: 在高负载下 RingBuffer 正常工作，无数据溢出或丢失。
- [ ] **批量提交**: 验证数据是否按阈值（数量/时间）批量提交。

## 6. 熔断与恢复 (Circuit Breaker)
- [ ] **点位熔断**: 模拟单点故障，验证是否进入 5 分钟冷却期。
- [ ] **设备降级**: 模拟设备高故障率，验证采集频率是否降低。

## 7. 监控指标 (Metrics)
- [ ] **指标暴露**: 验证 `/metrics` 端点是否有 Modbus 相关 Counter/Histogram。
- [ ] **健康度**: 验证设备状态（Online/Offline/Degraded）切换准确。

## 8. 性能测试 (Performance)
- [ ] **1000 点位**: CPU < 10%, 周期 < 500ms。
- [ ] **5000 点位**: CPU < 25%, 周期 < 2s。
- [ ] **10000 点位**: CPU < 40%, 周期 < 3s。
